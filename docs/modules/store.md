- Store 模块

Store 具有 2 项功能：

- 存储: 存储数据
- 通讯: 响应数据变动操作

大致定下的数据结构：

>  输入数据集合 ---> 行程原始数据集合 ----> 行程视图数据集合-----> 绘制数据集合

原始数据集合，为所有操作的 根本数据，所有操作都是通过操作原始数据集合来控制下方 的数据变动

约定规则： 父级属性，变动会导致子级属性的变动； 子级属性的变动，不会导致父级属性的变动

原始数据集合大概是这样的构成：

```
{
  width:100
  hidden:true,
  rows：[{
    //cell的基本信息
  },{}
  ,{}],
  merge:[],
  getStyle:()=>{}
}
```
